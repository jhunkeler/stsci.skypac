Pixel Area Map (PAM) utilities¶

A module that provides functions for computing Pixel Area Maps (PAM) based on distortion model contained in a FITS WCS.

Warning

At this moment, this is an experimental feature.

Authors:
 Mihai Cara (contact: help@stsci.edu)

stsci.skypac.pamutils.pam_from_file(image, ext, output_pam)¶

Generate a Pixel Area Map (PAM) file from the FITS WCS contained in an image extension of a calibrated HST image file specified by image.

Parameters:
 
image : str

File name of a FITS image that will provide a FITS WCS.

ext : int, str, tuple of (str, int)

Extension specification. May be an integer extension number, a string extension name, or a tuple of extension name and extension version.

output_pam : str

Output file name to which PAM will be written.

Warning

If the output file already exists, it will be overwritten without warnings.
stsci.skypac.pamutils.pam_from_wcs(wcs, shape=None)¶

Generate a Pixel Area Map (PAM) file from a FITS WCS.

Parameters:
 
wcs : astropy.wcs.WCS

An WCS object to be used for generating PAM file.

shape : tuple of two int, None, optional

Shape of the output image (ny, nx). If se to default None, this function will try to deduce the shape of the output image from the values of _naxis1 and _naxis2 attributes of the input wcs object.
Returns:
 
pam : numpy.ndarray

A 2D numpy.ndarray containing PAM.
Raises:
 
ValueError

When shape is None but the wcs object has no _naxis1 and _naxis2 attributes.
